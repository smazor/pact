### 11) What happens when system prompts are the only enforcement layer?

We ran a live demonstration against OpenClaw, a real open-source AI agent platform. Two agents coordinate dinner plans via a shared gateway using real WebSocket connections, ed25519 device authentication, and real LLM inference. An attacker sends a single chat message containing a hidden boundary marker that tricks the LLM into treating injected text as owner instructions. The agent forwards a fraudulent payment request, a $2,000 hotel booking, and a cover message to the other agent. All three succeed. The system prompt provides no resistance because the LLM cannot distinguish owner instructions from injected instructions in the same context window. Research consistently confirms near-100% bypass rates for indirect prompt injection across all major LLMs.
Now transpose this to a real enterprise setting. A procurement agent is negotiating a $250,000 rush order for replacement pumps — approved vendors only, delivery by Friday. A poisoned product description in a vendor catalog contains the same kind of injection: hidden text that instructs the agent to issue a purchase order to an unapproved vendor at an inflated price. The attack is structurally identical. The only difference is the dollar amount.

With a Vincul coalition contract governing the workflow, the compromised agent attempts the same actions but every tool call hits the 7-step enforcement pipeline. Unapproved vendor? Denied at step 5 (predicate). Over budget? Denied at step 6 (ceiling). Wrong destination? Denied at step 4 (namespace). Each denial produces a cryptographic receipt — proof of what was attempted, under which scope, and why it was blocked — before any irreversible commitment was made. The authority does not come from a system prompt the LLM can be tricked into ignoring. It comes from a coalition contract that both parties agreed to before any agent acted.
